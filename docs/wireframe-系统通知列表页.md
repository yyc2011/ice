## 模块说明

### 通知条目结构

| 字段 | 说明 |
|---|---|
| 图标 | 固定系统图标 [📢]，无用户头像 |
| 通知标题 | 通知类型名称，见下方分类表 |
| 通知正文 | 关联内容标题 + 核心信息（通过/未通过原因/公告摘要） |
| 操作链接 | 部分通知带快捷操作按钮（如「修改文章后重新提交」「申请复审」） |
| 时间 | 遵循 G2 规范 |

### 通知类型与跳转目标

| 通知类型 | 触发时机 | 跳转目标 |
|---|---|---|
| 文章审核通过 | 文章经 AI 或人工审核通过后 | 文章详情页 |
| 文章审核未通过 | 文章审核拒绝后 | 写文页（带原文内容，可修改重提）；或消耗书币申请 AI/人工复审 |
| 话题审核通过 | 高等级用户发起话题审核通过后 | 话题详情页 |
| 话题审核未通过 | 话题审核拒绝后 | 无跳转，仅展示原因 |
| 系统公告 | 管理员发布全站公告 | 原地展开全文 |

### 申请复审流程（文章审核未通过通知内）
- 通知卡片内展示两个按钮：「AI复审 10书币」「人工复审 30书币」
- 点击任一按钮 → 弹出申诉说明输入框（必填，描述合规理由）
- 确认后扣除书币，进入对应复审队列；书币不退（服务费）
- 余额不足时按钮禁用，引导跳转书币充值页
- 规则见 [`05-track-书币系统-2026-05-05.md`](05-track-书币系统-2026-05-05.md) 第六章

### 排序规则
- 按通知时间倒序排列，全量加载（分页滚动）
- 进入页面后，所有条目标记为已读，消息中心页「系统通知」角标清零

### 空态
无任何系统通知时显示：「暂无系统通知」
