## 一、约定

### 1.1 基础

| 项 | 值 |
|---|---|
| Base URL | `https://api.example.com`（生产）/ `http://localhost:8080`（本地） |
| 协议 | HTTPS（生产） |
| 格式 | JSON，`Content-Type: application/json` |
| 认证 | Header `Authorization: Bearer {jwt}`（需登录接口） |
| 分页 | `?page=1&size=20`；响应 `{ items, total, page, size }` |
| 错误体 | `{ "code": "ARTICLE_NOT_FOUND", "message": "..." }` |

### 1.2 与前端联动

- 所有**对外**接口由 `server/` SpringDoc 生成 `openapi/openapi.json`
- 小程序与管理后台均通过根目录 `npm run generate:api` 生成 TypeScript 客户端
  - 小程序输出：`miniprogram/api/generated/`（含 wx 兼容补丁）
  - 管理后台输出：`admin/src/api/generated/`
- 页面通过各端 `api/instances.ts` 调用生成的 `*ControllerApi` 方法，禁止手写 `fetch` / `wx.request` 拼路径
- **禁止**小程序直连 `ai/` 服务
- 小程序启动后先调 `GET /api/v1/config/client` 获取功能开关（如充值入口显隐）

### 1.3 能力开关与 Stub 约定

| 能力 | 接口 | MVP 实现 |
|---|---|---|
| 微信支付充值 | §2.8、附录 D | **契约保留**；`recharge_enabled=false` 时隐藏入口；底层 `WechatPayClient` **留空**（待企业资质） |
| AI 内容审核 | §五、[`12-track`](12-track-审核模块-2026-07-08.md) | `server` → `ai/internal/audit`；`ai/` 多 Provider 可插拔，默认 mock |

### 1.4 文章状态枚举

| 值 | 含义 |
|:---:|---|
| `draft` | 草稿 |
| `reviewing` | 审核中 |
| `published` | 已发布 |
| `rejected` | 已拒绝 |
| `deleted` | 已删除 |

### 2.11 客户端配置 `GET /api/v1/config/client`

无需登录；小程序 `onLaunch` 与「我的」页 `onShow` 时拉取，缓存至本地。

**响应 200**

```json
{
  "features": {
    "recharge_enabled": false,
    "recharge_custom_amount_enabled": false
  },
  "recharge_ratio": 10,
  "version": "2026-07-08"
}
```

| 字段 | 说明 |
|---|---|
| `features.recharge_enabled` | `true` 时展示：书币明细「充值」、余额不足引导充值、写文/话题等场景的充值跳转 |
| `features.recharge_custom_amount_enabled` | `true` 时充值页展示「自定义金额」档位 |
| `recharge_ratio` | 1 元 = N 书币，与 `book_coin_config` 同步 |

**前端显隐规则**（与线框对齐）：

| 入口 | `recharge_enabled=false` | `recharge_enabled=true` |
|---|---|---|
| 书币明细页右上角「充值 →」 | 隐藏 | 展示 |
| 我的页「书币明细」旁无独立充值钮 | — | 经明细页进入 |
| 私信/写文/话题 余额不足引导 | 仅提示「书币不足」，**不展示**「去充值」 | 展示「去充值书币 →」 |
| 书币充值页路由 | 不可达（入口已隐藏）；直链访问时展示「充值功能暂未开放」 | 正常 |

## 三、审核相关接口（server 编排）

> 业务规则见 [`12-track-审核模块-2026-07-08.md`](12-track-审核模块-2026-07-08.md)。

### 3.1 发布触发审核（内部流程，非独立 HTTP）

`POST /api/v1/articles/{id}/publish` 与 `POST /api/v1/topics` 在 server 内执行：

```
1. 校验权限与字段
2. 写 article/topic.status = reviewing
3. 插入 content_review（review_type=首次AI, status=待处理）
4. 同步调用 ai POST /internal/audit（15s 超时）
5a. AI 通过 → 更新 status=published/ongoing，写 ai_quality_score，发通知，奖书币
5b. AI 拒绝 → status=rejected，写 reject_reason，发通知
5c. 超时/5xx → 保持 reviewing，content_review 待处理，进人工队列
```

### 3.2 修改重提

`PUT /api/v1/articles/{id}` + `POST /api/v1/articles/{id}/publish`（`status=rejected` 时）：

- 不扣书币
- 重新走 §3.1 流程（`review_type=首次AI`）

### 3.3 申请复审

`POST /api/v1/articles/{id}/review-appeal`

**请求体**

```json
{
  "appeal_type": "ai",
  "appeal_text": "本文为个人真实经历，无广告引流..."
}
```

| `appeal_type` | 书币 | 行为 |
|---|---|---|
| `ai` | 10 | 扣费 → AI 复审 → 同步结果 |
| `human` | 30 | 扣费 → 人工队列 → 异步通知 |

**响应 202**

```json
{
  "review_id": 5001,
  "status": "reviewing",
  "message": "已提交AI复审"
}
```

人工复审返回 `status: "pending_manual"`，完成后推送通知。

### 3.4 查询审核状态

`GET /api/v1/articles/{id}` 响应含：

```json
{
  "id": 1001,
  "status": "rejected",
  "reject_reason": "含有违反社区规范的内容",
  "latest_review": {
    "review_type": "initial",
    "ai_score": 45,
    "completed_at": "2026-07-08T14:00:00Z"
  }
}
```

## 五、内部 AI 服务接口

> **不对小程序暴露**。审核契约见 [`12-track` 附录 A](12-track-审核模块-2026-07-08.md)；**多模型 Provider 可插拔**，见 §5.2 与附录 E。

### 5.1 HTTP 端点

| 方法 | 路径 | 调用方 |
|---|---|---|
| GET | `/health` | Docker 探活 |
| POST | `/internal/audit` | `server/` 审核编排 |
| GET | `/internal/providers` | 运维/debug；返回已注册 Provider 列表与当前激活项 |

### 5.2 多模型 Provider 架构（`ai/`）

```
server ──POST /internal/audit──► ai/app/routers/audit.py
                                        │
                                        ▼
                                 ProviderRegistry
                    ┌──────────┬──────────┬──────────┬──────────┐
                    ▼          ▼          ▼          ▼          ▼
                 mock    deepseek    doubao      glm    langchain
                 (默认)   (直连API)  (火山引擎)  (智谱)  (编排层/路由)
```

| Provider code | 说明 | MVP 状态 |
|---|---|---|
| `mock` | 规则 Mock，不依赖外网 | **默认启用** |
| `deepseek` | DeepSeek 开放 API | 接口预留，实现留空 |
| `doubao` | 字节豆包 / 火山方舟 | 接口预留，实现留空 |
| `glm` | 智谱 GLM | 接口预留，实现留空 |
| `langchain` | LangChain 统一编排层，可路由至上述任一模型 | 接口预留，实现留空 |

- 环境变量 `AI_PROVIDER` 指定激活 Provider；未配置或 Key 缺失时回退 `mock`
- `langchain` 作为**可选编排层**，不替代 `/internal/audit` 对外契约；内部可将 Prompt/Chain 委托给 LangChain，再调具体模型
- `server/` **禁止**直连大模型；所有 AI 能力经 `ai/` 内网调用

server 配置项（`application.yml`）：

```yaml
ice:
  ai:
    base-url: http://localhost:8000
    internal-token: ${INTERNAL_API_TOKEN}
    timeout-ms: 15000
    # 仅文档/运维参考；实际 Provider 在 ai 服务环境变量配置
    provider: ${AI_PROVIDER:mock}
```

ai 服务配置项（`ai/.env`）见附录 E。
