## 模块说明

### 模块1 顶部搜索条
- 保留返回按钮，点击返回来源页面（通常为首页）。
- 搜索框可直接编辑关键词并重新搜索，减少来回跳转。
- 搜索类型支持：标题 / 作者 / 标签 / 话题；切换后立即请求对应单类型结果。

### 模块2 结果信息栏
- 展示当前关键词、当前搜索类型、结果总数。
- 结果总数由后端返回，前端仅展示，不做统计推导。

### 模块3 单类型结果列表
- 标题/标签搜索：仅返回文章卡片列表，结构为“标题单行 + 作者行 + 摘要与封面图同一行 + 底部元信息行”。
- 标题/标签搜索（无封面图）：去掉图片占位，摘要文本区域左右填充铺满卡片宽度。
- 标题/标签搜索中，作者头像/昵称为可点击入口，跳转个人主页。
- 作者搜索：仅返回用户卡片列表，不放“查看主页”按钮，整卡点击跳转用户主页。
- 话题搜索：卡片样式复用话题页「进行中话题」视觉风格（背景封面叠加标题与摘要），不放“去参与”按钮，整卡点击跳转话题详情页。
- 话题搜索（无封面图）：去掉封面展示区域，仅保留标题、摘录与统计信息的纯文本卡片。
- 话题搜索（已结束态）：不显示「⏳ 还剩N天」字段，改为显示「✅ 已结束 · 共N篇」；封面图、标题、摘录样式不变；整卡仍可点击跳转话题详情页。
- 不做混合结果流，降低认知负担。

### 模块4 排序与过滤策略
- 前端不提供过滤器与排序控件，保持页面简洁。
- 文章搜索结果默认即已发布集合，无“全部/已发布”切换。
- **排序已定** — 文章：已发布，按 `published_at` 降序；用户：昵称匹配优先，其次 `article_count` 降序；话题：进行中优先，按 `article_count` 降序

### 模块5 状态反馈
- 有结果：卡片流 + 加载更多。
- 无结果：提供弱引导（更短关键词、切换搜索类型）。
- 异常态：提供重试按钮，不阻断用户继续搜索。

### 搜索历史 / 搜索建议（本阶段不做）
- 搜索框聚焦或无关键词时展示「历史搜索」「热门搜索词」的功能，本阶段不做，后续再考虑。
- 当前进入搜索结果页直接展示结果，无历史/建议面板。
