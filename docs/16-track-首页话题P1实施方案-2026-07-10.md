## 三、算法落地（应用层）

### `hot_score`

```
hot_score = ln(1 + view_count) × w_view
          + like_count × w_like
          + comment_count × w_comment
          - dislike_count × w_dislike
```

权重从 `ranking_config` 读取。入榜资格：`status=published` 且 (`like_count >= hot_min_likes` OR `view_count >= hot_min_views`)。

### `featured_score`

```
freshness = max(0, 100 - hours_since_publish × 2)
featured_score = ai_quality_score × 0.7 + freshness × 0.3
```

近 7 天已发布文章；`ai_quality_score` 为空时用 70。

### 话题排序

| 场景 | 规则 |
|---|---|
| 首页模块1 / 话题模块2 进行中 | `status=1`，`article_count` 降序；同分按 `end_at` 升序（话题页）或 `start_at` 降序（首页） |
| 即将开始 | `status=0` 且 `start_at > now()`，按 `start_at` 升序 |
| 历史热门 | `status=2`，按 Tab 时间窗筛选后 `article_count` 降序 |

## 五、话题发起审核

复用 `ArticleService` 审核编排模式：

1. 校验 Lv2+（`UserLevel.canCreateTopic`）
2. 扣 20 书币（`BookCoinService.spend`）
3. 写 `topic.status=0`（审核中）
4. 插 `content_review`（`ContentType.TOPIC`）
5. 调 `AiAuditClient.auditTopic(...)`
6. 通过 → `status=1`（进行中），设 `start_at`/`end_at`
7. 拒绝 → `status=3`，退还发起费
