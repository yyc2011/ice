## 模块说明

### 模块1 搜索框
- **T1已定**：搜索框补编号为模块1，后续各模块依次为模块2-5
- 搜索范围：话题标题
- 点击搜索框 → 跳转搜索结果页（过滤维度为话题）

### 模块2 进行中的话题
- 同时进行中话题数量上限：5-10个（业务约束）
- 数量为 0 时，整个模块2不展示
- 卡片宽度约占屏幕90%，横向滑动浏览
- 卡片顺序：**排序已定** — `status=进行中`，按 `article_count` 降序，同分按 `end_at` 升序（临近截止优先）
- 卡片封面展示规则：
  - 有图片素材 → 图片作为背景，话题标题叠加在上方
  - 仅文字素材 → 纯色背景 + 素材摘录首1-2句
  - 仅标题 → 纯色背景 + 大号标题居中
- 卡片底部显示：参与篇数 · 剩余时间
- 点击「去参与」→ 跳转写文页（话题信息已预填）
- 点击卡片其他区域 → 跳转话题详情页

### 模块3 即将开始
- 展示已审核通过、尚未开始的话题
- 默认显示最近2条，点击「查看全部」展开所有
- 每条显示：话题标题 · 距开始时间
- 点击 → 跳转话题详情页（详情页内无法参与，仅供预览）

### 模块4 历史热门
- 三个Tab：今日 / 本月 / 本年，默认选中「今日」
- 每个Tab各展示10条历史（已结束）话题
- **排序已定** — 已结束话题按 Tab 时间窗筛选后，按该窗内 `article_count` 降序；同分按 `end_at` 降序
- 每条显示：排名序号 · 话题标题 · 参与篇数 · 总阅读量
- 点击 → 跳转话题详情页

### 模块5 随机发现
- **T2已定**：与模块4 Tab（今日/本月/本年）**独立，不联动**；切换Tab不影响本模块内容
- 来源：所有已结束的历史话题（不排除已出现在模块4中的）
- 每次进入页面随机抽取约6-8条，双列瀑布流展示
- 滚动到底部自动加载更多（再次随机抽取）
- 每张卡片显示：封面图（或纯色背景）+ 话题标题 + 「已结束」标签
- 点击 → 跳转话题详情页

### 悬浮按钮「+ 发起话题」
- **T3已定**：对所有登录用户显示，固定在页面右下角（FAB 样式）
- 点击后后端校验：Lv2+ 或管理员 → 跳转发起话题页；不足 Lv2 → 提示「达到成长等级后可发起话题」
- 按钮始终悬浮可见，不随页面滚动消失
