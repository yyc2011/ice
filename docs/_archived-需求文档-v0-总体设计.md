## 一、产品定位

面向社会普罗大众的**文字创作与阅读社区**微信小程序。

- **核心用户群**：城市白领、有情怀的农民工、文艺创作者
- **特殊服务群体**：小学/初中/高中学生及教师（需认证，含有效期）
- **产品气质**：接地气，给普通人一个写作、抒发情感的地方
- **界面风格**：列表页采用图文卡片（小红书风格），文章详情页采用沉浸式阅读（微信读书风格）

## 四、用户等级与书币体系

> **重要说明**：等级体系与书币体系已完全解耦。等级是基于创作行为（发文数量、好评、话题热榜等）的独立系统，书币是平台货币/消耗品。等级与发文长度上限的映射关系待定，另行讨论。书币完整规则见 [`05-track-书币系统-2026-05-05.md`](05-track-书币系统-2026-05-05.md)。

### 4.1 等级与文章长度权限（等级升级规则待定）

| 等级  | 名称   | 免费发文字数上限 | 覆盖用户比例 |
| --- | ---- | -------- | ------ |
| Lv1 | 新芽   | 800 字    | ~50%   |
| Lv2 | 成长   | 1500 字   | ~40%   |
| Lv3 | 创作者  | 3000 字   | ~5%    |
| Lv4 | 文豪   | 5000 字   | ~5%    |

> **待细化**：等级升级门槛（基于创作行为，非书币余额）另行专题讨论。

### 4.2 书币解锁超额发文

用户可消耗书币临时突破本等级字数限制，**仅对当篇文章有效**：

- 每消耗 **10 书币**，可将当篇文章字数上限提高 **500 字**
- 可多次累加购买，直至达到全站最高上限 **5000 字**（不可再突破）
- 举例：Lv1 用户（免费 800 字）想写 1600 字，需超出 800 字 → 购买 2 档 = **20 书币**

### 4.3 书币获取

| 来源 | 获得书币 | 每日上限 |
| --- | --- | --- |
| 充值（微信支付） | 1 元 = 10 书币 | 无上限 |
| 发表文章（审核通过） | +5 书币/篇 | 每日最多 3 篇计奖励 |
| 热榜奖励（24h Top 10） | 梯度：#1=50、#2~3=30、#4~10=10 | 每篇每自然日最多一次 |
| 话题奖励池 | 由话题发起人设定，话题结束后按排名发放 | — |

### 4.4 书币消耗

| 消耗场景 | 消耗书币 |
| --- | --- |
| 发起话题 | 20 书币 |
| 话题奖励池充值 | 自定义，最低 50 书币 |
| 差评文章 | 5 书币/次（差评者消耗；作者不扣） |
| 举报内容 | 10 书币/次（举报成立退还） |
| 申请复审（被拒文章） | AI 复审 10 书币 · 人工复审 30 书币（不退） |
| 发送私信（对方未关注我） | 5 书币/条 |
| 字数突破（当篇有效） | 10 书币 / 每增加 500 字 |
| 查看主页访客列表 | 5 书币/次 |

### 4.5 书币余额规则

- 书币余额最低为 0，不会为负数
- 余额为 0 时，所有消耗型操作被阻止，引导充值

## 五、文章模块

### 5.1 发文规则

- 字数上限由用户等级决定（见 4.1），可花书币临时突破（见 4.2）
- **发文频率限制**：每天最多发布 N 篇（超级管理员可配置，初始建议 **3 篇/天**）

### 5.2 文章内容构成


| 内容元素 | 是否必填 | 规则说明                             |
| ---- | ---- | -------------------------------- |
| 正文文字 | 必填   | 长度受等级限制                          |
| 标签   | 必填   | 最多 5 个，每个不超过 10 汉字               |
| 图片   | 可选   | 最多 3 张，展示于文章**顶部横向滑动**，不可插入正文    |
| 背景音乐 | 可选   | 本阶段已砍掉（P0-3 已定），后续迭代再考虑      |
| 封面图  | 可选   | 有图片时默认取第一张；无图片时可单独上传封面，否则显示默认占位图 |


### 5.3 文章审核流程

```
用户提交文章
    │
    ▼
AI 审核（DeepSeek / 豆包）
    │
    ├── 评分 ≥ 60 → 自动通过，公开发布
    │
    └── 评分 < 60 → 进入人工审核队列
                        │
                        ├── 管理员审核通过 → 公开发布
                        │
                        └── 管理员驳回 → 通知作者 + 填写驳回理由

用户也可主动选择"申请人工审核"，跳过 AI 直接进入人工队列
```

> **待细化**：AI 审核评分维度（合规性、内容质量、违禁词等）；DeepSeek 与豆包的选型需对比实际 API 调用成本后确定。

### 5.4 文章互动功能


| 功能  | 说明                                           |
| --- | -------------------------------------------- |
| 点赞  | 不涉及书币                                       |
| 差评  | 差评者消耗 5 书币；作者不扣书币                           |
| 评论  | 支持**引用文字块**（评论列表中显示引用段落，以 blockquote 语义标签实现） |
| 收藏  | 收藏至个人收藏夹                                     |
| 分享  | 转发给微信好友 / 分享到朋友圈 / 生成分享图片                    |

## 六、标签系统

### 6.1 架构选型：混合标签（Hybrid）+ 话题双轨制

产品内有两种独立的"分类"概念，**不混用**：


| 类型            | 性质        | 示例           | 架构           |
| ------------- | --------- | ------------ | ------------ |
| **标签（Tag）**   | 永久，文章内容属性 | 散文、诗歌、情感、日记  | 混合标签（Hybrid） |
| **话题（Topic）** | 限时，活动聚合   | 高考作文季、春天写作话题 | 独立活动模块       |


### 6.2 标签规则

- 管理员预设核心标签（散文、诗歌、随笔、日记、情感、成长、记录……）
- 用户发文时可从预设中选择，也可自定义创建（≤ 10 汉字）
- 发布时系统做**精确匹配去重**：输入"散文"若已存在则复用，不新建
- 一篇文章最多绑定 **5 个标签**
- 未来可升级为模糊匹配（如"散文体"自动合并到"散文"）

### 6.3 标签数据库设计（核心表）

```sql
-- 标签主表
CREATE TABLE tag (
    id          BIGINT PRIMARY KEY AUTO_INCREMENT,
    name        VARCHAR(30)  NOT NULL UNIQUE,   -- 标签名，最多10汉字
    source      TINYINT      NOT NULL,          -- 0=管理员预设 1=用户创建
    creator_id  BIGINT,                         -- 创建者用户ID（管理员创建时为NULL）
    article_count INT DEFAULT 0,               -- 关联文章数（冗余字段，用于展示）
    status      TINYINT DEFAULT 1,             -- 0=下架 1=正常 2=待审核
    created_at  DATETIME     NOT NULL
);

-- 文章-标签多对多关联表
CREATE TABLE article_tag (
    article_id  BIGINT NOT NULL,
    tag_id      BIGINT NOT NULL,
    PRIMARY KEY (article_id, tag_id)
);
```

参考微信读书 + 小红书首页风格，自上而下：


| 区块       | 内容                  | 形式            |

## 九.1、我的页（Tab）重构定义

“我的”页定位为用户个人管理台，优先满足内容管理与账号安全，不再以社交信息堆叠为主。

### 9.1 页面模块顺序（固定）

1. 个人资料卡
2. 文章管理（纵向）
3. 我发起的话题
4. 设置与账号安全

### 9.2 个人资料卡

- 展示字段：头像、昵称、账号ID、书币余额（仅展示位，具体实现后置）
- 操作入口：登录/退出登录（按登录态二选一）、编辑资料、书币明细（可先占位）

### 9.3 文章管理（纵向分组规则）

- 若存在分组：先按分组纵向展示组内文章，再展示“未分组文章”
- 若不存在分组：仅展示“未分组文章”
- 分组能力：新建分组、重命名、排序、删除组（删除组不删除文章，文章回到未分组）
- 文章能力：批量移动文章到指定分组/未分组
- 检索能力：支持按标题搜索、按状态筛选（草稿/已发布/审核中）

### 9.4 我发起的话题

- 默认列表：我发起的话题，按最近更新时间倒序
- 可切换查看：我参与的话题（增强回访效率）
- 条目字段：话题标题、状态（进行中/已结束）、参与文章数、最后更新时间

### 9.5 设置与账号安全

- 设置页统一承载：手机号绑定、微信号绑定、昵称修改、登录密码修改
- 预留账号注销入口，需二次确认

## 十、学生/教师认证

### 10.1 认证流程

1. 用户申请认证，选择身份类型，填写入学/任职年份
2. 上传学生证/教师资格证照片
3. 管理员在后台审核（通过/驳回）

### 10.2 自动过期规则


| 身份  | 过期时间                 |
| --- | -------------------- |
| 小学生 | 入学年份 + 6 年           |
| 初中生 | 入学年份 + 3 年           |
| 高中生 | 入学年份 + 3 年           |
| 教师  | 认证通过日期 + 3 年，到期可重新认证 |


- 过期后身份标识自动消失，用户降回普通用户
- 系统每日定时任务检查并处理过期认证

> **暂缓功能**：教师针对学生的学习记录模块，后续单独规划。

## 十三、技术选型

### 13.1 前端（微信小程序）


| 技术    | 选择                                      |
| ----- | --------------------------------------- |
| 开发语言  | TypeScript                              |
| UI 框架 | Vant Weapp                              |
| 接口客户端 | openapi-generator 自动生成（基于后端 OpenAPI 规范） |


### 13.2 前后端接口联动方案

- 后端使用 **SpringDoc** 自动生成 `openapi.json`
- 使用 **openapi-generator-cli** 生成 TypeScript fetch 客户端
- 小程序引用生成的客户端代码调用接口
- 接口变动时重新生成，编译期报错，避免前端调用废弃接口

### 13.3 后端


| 技术   | 选择                      | 说明                                                  |
| ---- | ----------------------- | --------------------------------------------------- |
| 语言   | **Kotlin**              | 空安全、Data Class、协程，比 Java 更简洁；Spring Boot 3.x 一等公民支持 |
| 框架   | Spring Boot 3.x         |                                                     |
| ORM  | JOOQ                    | 类型安全的 SQL DSL，代码生成操作数据库                             |
| 数据库  | MySQL 8                 |                                                     |
| 缓存   | Redis                   | 热点数据、书币计数、发文限流                                      |
| 依赖管理 | Gradle（Kotlin DSL）      |                                                     |
| 本地开发 | Docker + Docker Compose | 一键启动 MySQL + Redis，无需本地手动安装                         |


### 13.4 云服务（推荐腾讯云）

微信小程序项目首选腾讯云（与微信生态原生集成）：


| 服务       | 用途                |
| -------- | ----------------- |
| COS 对象存储 | 图片、音频文件存储，支持小程序直传 |
| CDN      | 静态资源加速            |
| 短信服务     | 手机号注册验证码          |
| 微信支付     | 书币充值（后期接入）        |


### 13.5 AI 内容审核

- **首选**：DeepSeek API（成本低，开放 API）
- **备选**：豆包（字节跳动）
- 审核策略：提交文章正文 → 大模型返回合规评分 → ≥ 60 自动通过，< 60 进人工队列

## 十六、关键风险提示


| 风险       | 说明                                  |
| -------- | ----------------------------------- |
| 微信内容审核政策 | 内容类小程序需通过微信审核，需提前准备 ICP 备案和内容安全合规方案 |
