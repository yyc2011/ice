## 1. 产品概述

微信小程序，文字创作与阅读社区。气质接地气，参考微信读书（沉浸阅读）与小红书（图文卡片信息流）。

**底部 Tab（已定）**：首页 · 话题 · ✏️写文 · 分类 · 我的

---

## 2. 核心术语

| 术语 | 定义 |
|---|---|
| 话题 | 限时写作题目；Lv2+ 可发起（消耗书币）；发文可选关联 |
| 标签 | 用户扁平标注；发文必填 1-5 个；可搜索新建 |
| 分类 | 管理员两级分类；发文选填；未选归入默认分类「综合 > 生活随笔」 |
| 书币 | 平台虚拟货币，1元=10书币；规则见 05-track |
| 用户等级 | 创作行为积累，与书币解耦；规则见 06-track |
| 精选新文 | 首页模块3；近7天发布，`featured_score` 排序（附录 A） |
| 24小时热榜 | 首页模块2；过去24h内发布，`hot_score` 排序（附录 A） |

## 4. 用户与等级

### 4.1 等级体系（06-track）

6 级：读者(Lv0) → 新芽(Lv1) → 成长(Lv2) → 创作者(Lv3) → 深耕(Lv4) → 妙笔(Lv5)

- 免费发文字数：800 / 800 / 1500 / 3000 / 4000 / 5000
- 注册即 Lv0 可发文（800 字）；首篇审核通过升入 Lv1
- 升级：**篇且赞硬门槛**（每级均有）；**Lv3 起**另须加权总分达标；热榜为加分项（占所需总分最多 25%，非必须）；详见 06-track 第三~五章
- 不降级；违规走封禁
- 展示：我的/个人主页昵称旁「图标+等级名」；文章卡仅等级名

### 4.2 权限

| 能力 | 最低等级 |
|---|---|
| 阅读/点赞/收藏/评论 | Lv0 |
| 发文 | Lv0 |
| 发起话题 | Lv2 |
| 书币字数突破 | Lv0~Lv4（Lv5 已达 5000 上限） |

## 5. 书币系统（05-track 摘要）

**获取**：充值(1:10)、发文奖励(+5/篇，日限3篇)、热榜奖励、话题奖励池

**消耗**：
| 场景 | 书币 |
|---|---|
| 发起话题 | 20 |
| 话题奖励池 | ≥50（自定义） |
| 差评 | 5 |
| 举报 | 10（成立退还） |
| AI/人工复审 | 10 / 30 |
| 私信（对方未关注） | 5/条 |
| 字数突破 | 10/+500字 |
| 查看访客列表 | 1/次（阶段二） |

字数突破：仅当篇有效，上限 5000 字。

## 6. 页面功能需求

### 6.1 首页
- 模块1：进行中话题横滑 5 个 + 「查看全部话题」
- 模块2：热榜 Top5 + 「查看完整榜单」
- 模块3：精选新文 + 「查看更多新文」→ 精选新文列表页
- 模块4：5 个推荐大类各预览 5 篇 + 「进入全部分类」
- 模块5：关注动态（阶段二占位，本阶段不展示）

### 6.2 写文页
- WYSIWYG 编辑器，字体与详情页一致
- 标签必填(1-5)、分类选填、话题可选、插图最多3张
- 封面：有插图默认首图；可自定义封面
- 字数：达等级免费上限后禁止新增输入，展示书币突破入口
- 发布触发 AI 审核

### 6.3 文章详情页
- 顶栏：返回、点赞、收藏、分享、⋯
- 分享：微信好友/朋友圈/生成图片
- ⋯面板：差评(消耗书币)、更多(举报，消耗书币)
- 评论：文后内联；长按正文可引用评论

### 6.4 话题
- 话题页：搜索、进行中/即将开始/已结束/随机发现；FAB 发起话题
- 发起话题页：标题/描述/时长/封面/奖励池；扣 20 书币 + 奖励池
- 话题详情：文章列表、统计、分享；投票阶段二

### 6.5 分类
- 分类页：两级目录纵向展开；搜索点击按钮触发
- 分类文章列表：面包屑可锚定返回；热度/时间排序

### 6.6 我的
- 资料卡：等级、书币、消息铃铛、关注/粉丝/文章/收藏数字行
- 文章管理：分组、草稿/发布/审核中筛选
- 我发起/参与的话题
- 编辑资料、设置（账号安全）

### 6.7 个人主页（对外）
- 资料区含等级、关注按钮、简介（最多 1024 字）
- 文章Tab（分组+Top5预览）/ 发起的话题Tab
- 无书币、无设置、无私信直入口

### 6.8 消息
- 消息中心：统一会话列表（私信+互动聚合+系统通知）
- 私信：对方未关注消耗书币；未回复最多3条
- 系统通知：审核结果、书币到账、升级通知；拒绝可申请复审

### 6.9 书币
- 书币明细：流水列表 + 充值入口（`recharge_enabled=true` 时展示）
- 书币充值：微信支付预设档位；接口见 11-track 附录 D；**资质未就绪时入口隐藏**

## 8. 审核流程

> 完整规则见 [`12-track-审核模块-2026-07-08.md`](12-track-审核模块-2026-07-08.md)；数据表见 [`08-track`](08-track-数据表设计-2026-07-07.md) 审核域。

1. **文章/话题发布** → 状态变为「审核中」→ 同步调用 AI 服务（`ai/internal/audit`，15s 超时）
2. **AI 通过**（综合分 ≥ 60）→ 文章「已发布」/ 话题「进行中」；写入 `ai_quality_score`；系统通知；发文奖书币
3. **AI 拒绝** → 「已拒绝」+ 原因；系统通知；用户可修改重提（免费）或消耗书币复审（AI 10 / 人工 30）
4. **AI 超时/故障** → 保持「审核中」，自动进人工队列
5. **话题拒绝** → 退还发起费 + 奖励池；不支持书币复审
6. **人工后台** → 管理员 Web 审核队列（见 [`13-track-管理后台MVP`](13-track-管理后台MVP-2026-07-08.md)）
7. **热榜结算** → 每日 0 点由 server 定时任务取 24h Top10 发奖（非实时接口）

## 附录 A：排序与热度算法

> 热榜、分类「热度」Tab、首页模块4大类预览共用 `hot_score`；精选新文使用 `featured_score`。配置项见 [`08-track`](08-track-数据表设计-2026-07-07.md) `ranking_config`。

### A.1 统一热度分 `hot_score`

```
hot_score =
  ln(1 + view_count) × 10
  + like_count × 3
  + comment_count × 10
  - dislike_count × 2
```

| 项 | 权重 | 说明 |
|---|---|---|
| 阅读 | `ln(1+阅读)×10` | 对数压缩，避免纯刷阅读霸榜 |
| 点赞 | `×3` | 与 06-track 等级「1分/赞」同量级思路 |
| 评论 | `×10` | 与话题奖励池「评论10分/条」一致 |
| 差评 | `×(-2)` | 负向信号，幅度小于评论 |

- **阅读量**：MVP 使用文章累计 `view_count`；阶段二可改为时间窗内增量统计
- **入榜资格**（须同时满足）：状态=已发布且审核通过；`hot_score > 0`；`like_count ≥ 1` **或** `view_count ≥ 20`
- 无足够文章时榜单可少于 10 条

### A.2 热榜时间窗（发布窗）

| Tab | 时间窗 | 排序 | 书币奖励 |
|---|---|---|---|
| 24小时 | 过去 24h 内**发布** | `hot_score` 降序 | **有**（05-track 每日 0 点 Top10） |
| 7天 | 过去 7 天发布 | 同上 | 无 |
| 30天 | 过去 30 天发布 | 同上 | 无 |

- 每日 0 点取「24小时 Tab」当前 Top10 发奖，与展示同源
- 首页模块2：展示 24h Top5，与热榜页 24h Tab 同源

**阶段二扩展（互动窗）**：可新增独立列表（如「正在热议」），`window_mode=engagement`，复用 `hot_score` 公式与窗内增量统计；**不改变**现有 24h 热榜与书币规则；新互动窗列表默认不发书币

### A.3 精选新文 `featured_score`

**范围**：已发布且审核通过；近 7 天内发布；首页固定 10 篇，精选新文列表页分页同源

```
freshness = max(0, 100 - hours_since_publish × 2)   // 0~48h 内从 100 线性衰减到 4
featured_score = ai_quality_score × 0.7 + freshness × 0.3
```

| 字段 | 说明 |
|---|---|
| `ai_quality_score` | AI 审核通过时产出，0–100；MVP 无 AI 细分时可用固定分 70 + 人工规则加减 |
| `freshness` | 鼓励新文，48h 后权重很低 |

精选偏「质量 + 新」，热榜偏「互动热度」；同一篇文章可同时出现在两个模块。

### A.4 分类列表「热度」与首页模块4

- **分类文章列表「热度」Tab**：该分类下已发布文章，近 30 天发布，按 `hot_score` 降序；「时间」Tab 为 `published_at` 倒序
- **首页模块4大类预览**：每个推荐大类（含下属小类）近 30 天、`hot_score` Top5

## 附录 B：举报原因与默认分类

### B.1 默认分类

发文未选分类时，自动归入：

| 层级 | 名称 | 说明 |
|---|---|---|
| 大类 | **综合** | 系统内置，不可删除 |
| 小类 | **生活随笔** | 默认归入；展示路径 `综合 > 生活随笔` |

用户可在「我的」文章管理中后续修改分类。

### B.2 举报原因枚举

与文章详情页、05-track（10 书币/次，成立退还）对齐。单选 + 可选补充说明：

| code | 展示文案 | 说明 |
|---|---|---|
| `illegal` | 违法违规 | 涉政、暴恐等 |
| `porn` | 色情低俗 | |
| `spam` | 垃圾广告 | 引流、重复灌水 |
| `plagiarism` | 抄袭洗稿 | |
| `abuse` | 人身攻击辱骂 | |
| `misinfo` | 不实信息 | |
| `other` | 其他 | 选后**必填**补充说明（10~200 字） |

**交互**：点「举报」→ 原因列表（单选）→ 若 `other` 则输入框 → 书币确认（10 书币）→ 提交；后台按 `reason` 筛选；成立则退书币并通知举报人。评论举报复用同一枚举，`target_type=comment`。
