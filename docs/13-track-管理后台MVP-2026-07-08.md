## 一、定位

| 项 | 说明 |
|---|---|
| 形态 | **独立 Web 管理后台**（浏览器访问），非小程序内入口 |
| MVP 范围 | 人工审核队列、分类 CRUD、举报处理、系统公告 |
| 不做 | 书币配置专用 UI、用户封禁批量工具、数据大盘（阶段二） |

---

## 二、权限模型

沿用 [`08-track`](08-track-数据表设计-2026-07-07.md) `user.role`：

| role（值越小权限越高） | 值 | API slug | 权限 |
|---|:---:|---|---|
| 超级管理员 | 0 | `super_admin` | 全部后台能力 + 阶段二增删管理员 |
| 管理员 | 1 | `admin` | 审核队列、举报、公告、分类 CRUD、功能开关 |
| 普通用户 | 2 | `user` | 无后台访问（微信首登默认） |

- 登录（MVP）：`POST /api/v1/auth/admin/login`（`login_name` + `password`）→ JWT；`UserRole.canAccessAdmin()` 校验 `role IN (0,1)`
- 开发联调：保留 `POST /api/v1/auth/dev/login`（仅非生产）
- 种子超管：Flyway V10 写入 `login_name=admin`、`role=0`；密码来自 `ADMIN_SEED_PASSWORD`

## 三、页面与功能

### 3.1 人工审核队列 `/admin/reviews`

| 功能 | 说明 |
|---|---|
| 列表 | `content_review.status=0`；筛选 `content_type`、`review_type`；优先 `review_type=3`（人工复审） |
| 详情 | 正文/标题/标签；AI `ai_score` + `ai_dimensions`；用户 `appeal_text` |
| 通过 | 更新文章/话题状态；写 `reviewer_id`、`completed_at`；发系统通知 |
| 拒绝 | 必选 `reason_code`（见 12-track §7.2）；写 `reject_reason` |

### 3.2 分类管理 `/admin/categories`

| 功能 | 说明 |
|---|---|
| 大类 CRUD | `parent_id=0`；不可删除有子类或仍有文章的大类 |
| 小类 CRUD | 挂在大类下；默认分类「综合>生活随笔」不可删 |
| 首页推荐 | 勾选最多 **5 个大类**，设置 `is_home_recommended=1`、`home_sort_order` 1–5 |
| 种子导入 | 首次部署执行 Flyway seed（见 [`15-track`](15-track-分类种子数据-2026-07-08.md)） |

### 3.3 举报处理 `/admin/reports`

| 功能 | 说明 |
|---|---|
| 列表 | `report.status=0`；按 `reason` 筛选 |
| 详情 | 举报人、目标文章/评论、原因、补充说明 |
| 成立 | `status=1`；退还举报人 10 书币；通知举报人；视情况下架内容 |
| 不成立 | `status=2`；通知举报人；书币不退 |

### 3.4 系统公告 `/admin/announcements`

| 功能 | 说明 |
|---|---|
| 新建 | 标题 + 正文；保存草稿或立即发布 |
| 发布 | 写 `announcement`；批量生成 `notification.type=system` |
| 列表 | 已发布/草稿；不可编辑已发布公告（MVP 仅新建） |

### 3.5 功能开关 `/admin/features`

| Key | 说明 |
|---|---|
| `recharge_enabled` | 是否开放充值（需微信支付资质；默认 `false`） |
| `recharge_custom_amount_enabled` | 是否展示自定义充值金额 |

修改后小程序下次拉取 `GET /api/v1/config/client` 生效。

## 四、API 映射

详见 [`11-track-API设计`](11-track-API设计-2026-07-08.md) 第四节；MVP 需补全的 HTTP 方法：

| 资源 | GET | POST | PUT | DELETE |
|---|---|:---:|:---:|:---:|
| `/admin/reviews` | 列表/详情 | approve/reject | — | — |
| `/admin/categories` | 树/详情 | 新建 | 更新 | 删除（受限） |
| `/admin/reports` | 列表/详情 | 成立/不成立 | — | — |
| `/admin/announcements` | 列表 | 新建/发布 | — | — |

## 五、技术建议

- 前端：`admin/` — Vue 3 + Element Plus（已实现）
- 认证：复用 server JWT；`Login.vue` 用户名密码为主，DEV 一键登录为辅
- 部署：与 `server/` 同域或子域 `admin.example.com`
